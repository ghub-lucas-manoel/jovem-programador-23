Abaixo vou deixar alguns erros/pontos de melhoria que eu identifiquei no teu projeto:

1. Estrutura
A organização das pastas não está seguindo o que a gente trabalhou em sala (pastas node-modules, src).
Mesmo que você tenha diferentes páginas que não utilizem outros arquivos JS e CSS, recomendo que você crie as pastas de cada página para facilitar a questão de escalonamento do projeto e manutenção.

2. HTML
Nos teus arquivos, as referências dos arquivos js estão dentro da tag footer. Elas podem se manter ao final da tag body, mas fora do footer - mais precisamente após ele, para que o código fique mais bem organizado.
Em alguins dos arquivos, tem um pouco de código CSS dentro do HTML. Por mais que seja pouco, é interessante que esse código tenha seu próprio arquivo.
2.1. Index
A declaração do seu arquivo personalizado de CSS está antes da declaração do CSS do Bootstrap. Isso pode fazer com que algumas estilizações não sejam aplicadas na página.
O nome de alguns ids (ex. container1, container2) pode ser algo mais claro/específico sobre o que o elemento representa, por via de fácil entendimento do código.

3. CSS
Eu vi que dentro do style.css, você usou estilizações específicas dentro de um único arquivo. Nesse caso, você pode partir pra abordagem de criar um arquivo "global.css", onde você declara os estilos padrão de todas as páginas, e outros arquivos .css para páginas específicas/partes específicas. Assim, fica mais fácil de dar manutenção no código do css.

4. JS
Evita deixar código comentado dentro dos arquivos, pois cada linha de código é processamento a mais pro navegado, fora que isso pode confundir outros programadores que estejam dando manutenção no teu código.

5. Aplicação
Eu vi que você utilizou um esquema de spans pra fazer o ícone de sanduíche. Na página Index, ele ficou encostando em cima, não ficou centralizado na div. Você também pode pesquisar por ícones para utilizar um ícone de sanduíche, e utilizar o recurso de collapsing da navbar do Bootstrap para controlar a animação e espaçamento dos itens do menu.
Como você não declarou o arquivo do jQuery, o console aponta erro na rotina de envio de dados do formulário por email. Rotina essa inclusive que não é utilizada na página por não existir nela nenhum botão com o id indicado no código js.
Senti falta de um botão "index", "home", ou "início" no menu. Por mais que o Zago Planner na nav volte pro início, essa ação pode não ser intuitiva para alguns usuários, e eles se encontrem perdidos caso precisem voltar para a tela inicial.
